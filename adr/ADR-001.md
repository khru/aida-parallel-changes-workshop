# ADR-001

## Title

Usar arquitectura por capas con puertos y adaptadores

## Status

Accepted

## Context

El workshop necesita mostrar evolucion segura de contratos sin acoplar la capa HTTP a SQL Server.

## Decision

Se organiza la API en controllers, handlers de aplicacion, puertos de lectura y escritura, y adaptadores de infraestructura.

## Consequences

- mejora la separacion de responsabilidades
- permite sustituir adaptadores sin romper casos de uso
- facilita explicar inversion de dependencias durante el workshop
