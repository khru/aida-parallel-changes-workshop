# ADR-002

## Title

Persistencia con SQL Server, Dapper y FluentMigrator

## Status

Accepted

## Context

El objetivo didactico requiere SQL explicito para mostrar dual write, backfill y contract sin ocultar detalles de migracion.

## Decision

Se usa SQL Server como base, Dapper para acceso a datos y FluentMigrator para versionado de esquema.

## Consequences

- se evita Entity Framework en este repositorio
- las consultas y migraciones son visibles y auditables
- el alumnado puede seguir la evolucion del schema commit a commit
